A startup project useful to get started using DITA which has various customizations.

Project folder structure:

  • filters - contains DITAVAL Filter files used to profile the DITA content when publishing.
  • images - contains images used in the DITA project.
  • topics - contains all the DITA topics in the project.
  • reusables - contains various reusable components. For example in the topic "reusable_components.dita" you can add various DITA elements like notes, paragraphs, lists, list items which are reused in various places in the project. More about reusing DITA content..

Saving settings at Project Level. Most of the preference pages in the Preferences folder can be saved at project level. The following settings are already saved at project level:

  • The Editor / Spell Check preferences page is saved at project level with automatic spell check enabled.
  • The Editor / Spell Check / Dictionaries preferences page is saved at project level and all learned words are saved in the custom-settings/learned-words/en.tdi file. (TODO nu merge de fapt).
  • The Editor / Edit Modes / Author preferences page is saved at project level.
  • The DITA / Maps preferences page is saved at project level.

For checking terminology you can use the Oxygen Terminology Checker add-on. You can add terminology rules to the oxygen-term-checker folder. For example the Oxygen user manual uses a set of Vale rules to check for common terminology problems.

Framework customization

DITA editing can be customized by extending the current DITA framework customization. The framework extension defined in the custom-settings/dita-extension/dita-extension.exf extension framework script file.

You can customize the existing DITA new file templates my making changes to the custom-settings/dita-map-extension/templates/ and custom-settings/dita-extension/templates/ folders.

The content completion configuration extension file custom-settings/dita-extension/resources/cc_config_ext.xml allows you to customize various aspects of content completion. (TODO nu merge).

You can add custom Schematron validation rules in the Schematron file custom-settings/dita-extension/resources/customRules.sch.

You can add your own custom actions to the toolbars, menus, contextual menus by editing the custom-settings/dita-extension/dita-extension.exf extension file.

You can customize the visual editing by modifying the CSS custom-settings/dita-extension/css/custom.css.
