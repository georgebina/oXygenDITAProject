This is a sample DITA Project which imposes custom settings for DITA editing. When the project is opened in the Oxygen Project view it will impose a custom set of global options as well as an imposed custom DITA framework extension.

The project contains the following folder structure:

  • custom-settings - contains custom DITA and DITA Map framework extensions;
  • filters - contains DITAVAL Filter files used to profile the DITA content when publishing;
  • images - contains images used in the DITA project;
  • oxygen-term-checker - contains a terminology file with a rule;
  • publishing - contains all the DITA topics in the project;
  • reusables - contains various reusable components. For example in the topic "reusable_components.dita" you can add various DITA elements like notes, paragraphs, lists, list items which are reused in various places in the project. More about reusing DITA content.;
  • topics - contains all the DITA topics in the project.

Most of the preference pages in the Preferences folder can be saved at project level. When the project is opened in the Project view, the following global options are imposed:

  • The Editor / Spell Check preferences page is saved at project level with automatic spell check enabled.
  • The Editor / Spell Check / Dictionaries preferences page is saved at project level and all learned words are saved in the custom-settings/learned-words/en.tdi file.
  • The Editor / Edit Modes / Author preferences page is saved at project level.
  • The DITA / Maps preferences page is saved at project level.

DITA editing can be customized by extending the current DITA framework customization. The framework extension is defined in the custom-settings/dita-extension/dita-extension.exf extension framework script file. You can add your own custom actions to the toolbars, menus, contextual menus, add, change or remove transformation scenarios, validation scenarios.

You can customize the existing DITA file templates my making changes to the and custom-settings/dita-extension/templates/ and custom-settings/dita-map-extension/templates/ folders.

The content completion configuration extension file custom-settings/dita-extension/resources/cc_config_ext.xml allows you to customize various aspects of content completion.

You can add custom Schematron validation rules in the Schematron file custom-settings/dita-extension/resources/customRules.sch.

You can also customize the visual editing by modifying the CSS custom-settings/dita-extension/css/custom.css.

For checking terminology you can use the Oxygen Terminology Checker add-on. You can add terminology rules to the oxygen-term-checker folder. For example the Oxygen user manual uses a set of Vale rules to check for common terminology problems.
